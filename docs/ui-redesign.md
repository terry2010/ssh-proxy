# termfast 界面重构方案

> 版本：v1.0
> 日期：2026-07-13
> 状态：设计草案

---

## 1. 概述与目标

### 1.1 当前状态

termfast 目前已经完成了核心功能开发，包括：

- 多服务器 SOCKS5/HTTP 代理管理
- SSH 连接状态监控与重连
- 触发器（Triggers）自动化脚本
- 模板库（内置脚本模板）
- 设置面板

但从界面截图来看，**功能可用性与视觉完成度之间存在明显落差**。用户虽然能找到入口，但会在以下环节产生认知摩擦：

- 标题栏塞入了实时流量，不符合 macOS 用户习惯
- 侧边栏操作按钮堆叠，缺少主次
- 红色 × 按钮语义危险，容易误操作
- 中英文、大小写混用，显得不专业
- 设置面板字段冗长，缺少分类导航

### 1.2 重构目标

本次重构不是“换皮肤”，而是**重新建立信息架构与交互语义**，目标为：

1. **清晰（Clarity）**：每个控件的用途一眼可辨，状态、操作、信息分层明确。
2. **一致（Consistency）**：文案、色彩、间距、字体、交互模式统一。
3. **高效（Efficiency）**：高频操作（连接/断开/查看日志/编辑触发器）路径最短。
4. **可信（Trustworthiness）**：删除、断开等不可逆/影响大的操作必须有明确反馈，不误导。

### 1.3 适用范围

本方案覆盖以下界面：

- 主窗口（侧边栏 + 详情区 + 日志抽屉）
- 服务器详情页（连接、代理、触发器、Auth 四个标签页）
- 模板库弹窗
- 设置弹窗/面板
- 空状态、错误状态、加载状态

## 2. 设计原则

### 2.1 信息层级优先

界面不是功能列表的平铺，而是信息的结构化呈现。所有元素按重要性分为三级：

| 层级 | 作用 | 示例 |
|------|------|------|
| L1 主要信息 | 用户当前最关注的内容 | 当前服务器名称、连接状态 |
| L2 辅助信息 | 帮助用户理解上下文 | 端口、流量、日志摘要 |
| L3 操作与元数据 | 用户可执行的动作 | 编辑、删除、手动触发 |

### 2.2 色彩语义

- **主色（Primary）**：用于最高频正向操作（添加、连接、保存）。
- **成功色（Success）**：连接成功、任务完成。
- **警告色（Warning）**：重连中、异常但未失败。
- **危险色（Danger）**：删除、强制断开、严重错误。
- **中性色（Neutral）**：文本、边框、背景、次要按钮。

**禁止把危险色用于“关闭页面”或“断开连接”等中性操作**。

### 2.3 文案规范

- 面向中文用户时，**核心术语使用中文**，英文仅用于无法简短翻译的技术缩写。
- 事件类型统一为：`连接时`、`IP 变化时`、`进程结束时`、`端口关闭时`、`重连时`。
- 按钮文案使用动词 + 名词，避免歧义：`删除服务器` 优于 `删除`，`断开连接` 优于 `断开`。

## 3. 现状诊断

### 3.1 全局问题

#### 3.1.1 标题栏承载实时数据

当前标题栏显示：

```
termfast   1 个连接   ↑ 283 B/s   ↓ 47 B/s
```

**问题**：

- macOS 用户期望标题栏用于拖拽和显示应用名/文档名。
- 实时流量放在标题栏会被误以为是可点击控件，且缩短了可拖拽区域。
- 连接数/流量属于**状态信息**，应放在状态栏或侧边栏顶部，而非标题栏。

**重构方向**：

- 标题栏仅保留 `termfast`。
- 连接数、上下行流量迁移到**底部状态栏**或侧边栏顶部汇总区。

#### 3.1.2 红色按钮语义危险

[Image 2] 中“连接”按钮右侧直接放置一个红色 ×。

**问题**：

- 红色通常代表“删除/危险/不可逆操作”。
- 若该按钮表示“关闭详情页”，则颜色与语义严重不符。
- 若该按钮表示“断开连接”，则会造成用户紧张，不敢点击。

**重构方向**：

- 关闭/返回使用中性图标（`←` 或 `×` 用中性色）。
- 断开连接使用**描边按钮**或文字按钮 `断开连接`。
- 删除操作必须明确标为 `删除服务器`，并配合二次确认弹窗。

#### 3.1.3 中英文与大小写混用

当前界面存在大量中英文混用：

- 事件类型：`On Connect`、`On Ip Change`、`On Port Closed`
- 日志按钮：`logs.expand`
- 标签页：`Auth`

**问题**：

- 中英文混用破坏界面专业感。
- 英文大小写不一致：`On Ip Change` 应为 `On IP Change`。
- `logs.expand` 看起来是代码 key，不是面向用户的文案。

**重构方向**：

- 统一使用中文术语，见第 6 节“文案规范”。
- 若必须保留英文，统一大小写并经过校对。

### 3.2 主窗口 [Image 1]

#### 3.2.1 侧边栏操作缺少主次

侧边栏从上到下依次排列：

- `+ 添加服务器`（主按钮）
- `全部连接` / `全部断开`
- `模板库` / `设置`
- 状态统计 + 流量
- 服务器列表

所有操作处于同一层级，没有区分“高频全局操作”与“低频工具入口”。

**重构方向**：

- 主按钮保留 `+ 添加服务器`。
- `全部连接`、`全部断开`、`模板库`、`设置` 改为工具栏图标或文字链接，放在列表上方或底部。
- 状态统计与服务器列表分组，用分割线或背景色区分。

#### 3.2.2 服务器列表行信息松散

当前列表行：

```
● 3333
  192.168.50.100   :1082   [开关]
```

**问题**：

- 端口号写成 `:1082`，与 IP 分离，扫描困难。
- 状态点、开关、可能的类型徽章（如 `FW`）三者颜色竞争注意力。
- 开关远离服务器名称，增加误触风险。

**重构方向**：

- 改为：
  ```
  [●] 3333                              [开关]
      192.168.50.100:1082   [类型标签]
  ```
- 状态指示灯只保留一种语义色。
- 类型标签（如 `FW`）用中性/浅色小标签，不抢状态色。
- 开关放在行尾，与行等高，增大点击区域。

#### 3.2.3 日志区层级混乱

日志区当前包含：

```
all   all   搜索...   清除日志   logs.expand   关闭
```

**问题**：

- 两个 `all` 下拉框缺少标签，用户不知道在筛选什么。
- `logs.expand` 是英文 key。
- “关闭”按钮放在日志区右侧，语义上是关闭日志面板还是关闭弹窗不明确。

**重构方向**：

- 增加标签：`级别` `来源` `搜索`。
- `logs.expand` 改为 `展开`。
- 日志面板默认折叠，显示摘要 `日志 · 12 条`，点击展开。

### 3.3 服务器详情 - 触发器 [Image 2]

#### 3.3.1 头部操作区危险

红色 × 问题已在 3.1.2 节说明，此处不重复。

#### 3.3.2 触发器卡片信息层级平均

当前触发器卡片包含：

- 事件徽章（On Connect）
- 触发器名称（Test）
- 命令（ls）
- 三个按钮：手动触发、编辑、删除

所有元素权重相近，用户无法快速扫描。

**重构方向**：

- 采用**表格/列表**布局：
  | 事件 | 名称 | 命令 | 操作 |
  |---|---|---|---|
  | 连接时 | Test | `ls` | ▶ 编辑 删除 |
- 命令使用等宽字体 + 浅灰背景。
- “手动触发”使用播放图标按钮 ▶，减少文字按钮噪音。
- “删除”收到行末的 `⋯` 菜单中，不在列表内直接展示红色文字。

### 3.4 模板库弹窗 [Image 3]

#### 3.4.1 标题栏存在无关操作

标题栏按钮：`添加` `导入` `导出日志` `关闭`。

**问题**：

- `导出日志` 与模板库功能无关，破坏用户心智模型。
- 标题栏操作过多，视觉上拥挤。

**重构方向**：

- 标题栏仅保留：`+ 新建模板` `导入` `关闭`。
- `导出日志` 迁移到日志面板或设置面板。

#### 3.4.2 模板缺少预览

列表只显示模板名称和事件类型，看不到具体脚本内容。

**重构方向**：

- 列表项支持点击展开或右侧预览面板。
- 悬停时显示脚本摘要 tooltip。
- 操作按钮：`使用此模板`（主）、`编辑`（次）、`删除`（菜单）。

### 3.5 设置弹窗 [Image 4]

#### 3.5.1 缺少分类导航

所有设置堆叠在一个长弹窗内，用户需要滚动查找。

**重构方向**：

- 改为**左侧分类导航 + 右侧内容页**。
- 分类：通用、代理默认值、触发器默认值、通知、关于。

#### 3.5.2 表单项对齐不统一

输入框、下拉框、数字框宽度参差不齐，标签与控件距离不一致。

**重构方向**：

- 统一标签宽度，控件右对齐或左对齐。
- 同一页面内输入框宽度一致。

#### 3.5.3 缺少保存/取消按钮

截图未展示保存/取消操作，用户不确定修改是否立即生效。

**重构方向**：

- 底部固定操作栏：`取消` `保存`。
- 若采用实时保存，需在标题显示“已自动保存”提示。

## 4. 分屏重构方案

### 4.1 总体布局

建议采用 **“侧边栏 + 详情区 + 底部日志抽屉”** 的三层结构：

```text
+-------------------------------------------------------------+
|  termfast                                          [_][o][x] |  <- 标题栏：仅应用名
+------+------------------------------------------------------+
| 侧   |                                                      |
| 边   |  服务器详情区                                        |
| 栏   |  （连接 / 代理 / 触发器 / Auth）                      |
|      |                                                      |
|      |                                                      |
|      |                                                      |
|      |                                                      |
+------+------------------------------------------------------+
|  日志 · 12 条              [级别] [来源] [搜索] [展开] [清除] |  <- 底部状态/日志栏
+-------------------------------------------------------------+
```

#### 4.1.1 标题栏

- 只显示 `termfast`。
- 如果有未保存的配置变更，可在标题后加 `·` 提示，但不建议放实时数据。

#### 4.1.2 侧边栏

侧边栏宽度建议 **240px**，内部结构：

```text
+------------------+
| + 添加服务器     |  <- 主按钮，全程可见
+------------------+
| 全部连接 · 断开  |  <- 工具栏图标或文字链接
| 模板库 · 设置    |
+------------------+
| 已连接 1 / 异常 0|  <- 状态汇总
| ↑ 0 B/s ↓ 0 B/s  |
+------------------+
| ● 3333        [开关]
|   192.168.50.100:1082
|                  |
| ○ 测试节点    [开关]
|   192.168.50.100:1081
|                  |
| ● zhuikan FW [开关]
|   zhuikan.com:9999
+------------------+
```

**规则**：

- 服务器列表项高度 **48px**（含内边距）。
- 选中态用浅蓝背景 + 左侧 3px 主色竖线。
- 开关在行右端，与行等高。
- 类型标签（如 `FW`）放在名称右侧，使用中性色小标签，不是彩色徽章。

#### 4.1.3 主详情区

- 未选中服务器时显示**空状态**：
  - 插图/图标
  - 文案：`选择左侧服务器查看详情，或添加新服务器`
  - 主按钮：`+ 添加服务器`
- 选中后显示服务器详情，顶部为页头，下方为标签页。

#### 4.1.4 底部日志抽屉

- 默认折叠，仅显示摘要行：
  ```
  日志 · 12 条    ↑ 283 B/s  ↓ 47 B/s    [级别] [来源] [搜索] [展开] [清除]
  ```
- 点击“展开”后向上滑出日志列表，高度约占窗口 30%。
- 日志列表支持按级别、来源筛选，支持搜索高亮。

### 4.2 主窗口详细设计

#### 4.2.1 侧边栏

##### 4.2.1.1 添加服务器按钮

- 使用主色填充按钮，宽度与侧边栏内容区等宽，左右留白 12px。
- 图标 `+` 在左，文字在右。
- 始终可见，避免用户找不到入口。

##### 4.2.1.2 全局操作工具栏

- 放在添加按钮下方，用图标 + tooltip 表示：
  - `全部连接`（闪电图标）
  - `全部断开`（断开图标）
  - `模板库`（库图标）
  - `设置`（齿轮图标）
- 可用时高亮，不可用时置灰并禁用 tooltip 说明原因。

##### 4.2.1.3 状态汇总区

- 显示：
  ```
  已连接 1 / 异常 0
  ↑ 283 B/s  ↓ 47 B/s
  ```
- 使用小号字体（12px），颜色为次要文本色。
- 与服务器列表用 1px 分割线分隔。

##### 4.2.1.4 服务器列表项

每一行的结构：

```text
[●] 服务器名称                        [开关]
    192.168.50.100:1082   [类型标签]
```

**状态指示**：

| 状态 | 指示点颜色 | 说明 |
|------|------------|------|
| 已连接 | 绿色 | 正常代理中 |
| 连接中 | 黄色（闪烁） | 正在建立 SSH 通道 |
| 已断开 | 灰色 | 未启用 |
| 异常 | 红色 | 连接失败或重连耗尽 |

**类型标签**：

- `FW` 等类型标签使用浅灰背景 + 深灰文字，圆角 4px，字号 10px。
- hover 时显示完整含义 tooltip。

#### 4.2.2 空状态

当没有选中任何服务器时，主区域显示：

```text
[ 插图：服务器图标 ]

还没有选中服务器
在左侧列表选择一个服务器，或点击“添加服务器”开始。

[ + 添加服务器 ]
```

- 插图使用中性色，不抢眼。
- 按钮与文案间距 16px。

### 4.3 服务器详情页

#### 4.3.1 页头设计

```text
[← 返回列表]   3333                              [编辑] [断开连接] [⋯]
               192.168.50.100:1082  ·  已连接
```

- 左侧返回箭头用于折叠侧边栏或返回列表（响应式场景）。
- 标题使用 18px medium。
- 副标题使用 12px 次要色，显示地址和状态。
- 右侧操作：
  - `编辑`：进入编辑模式
  - `断开连接` / `连接`：根据状态切换，使用描边按钮
  - `⋯`：更多菜单，包含 `复制配置`、`删除服务器`、`诊断`

#### 4.3.2 标签页

标签顺序建议：

1. **连接**：SSH 连接参数、认证信息、测试连接按钮。
2. **代理**：本地监听端口、代理类型、测试 URL。
3. **触发器**：自动化脚本列表。
4. **Auth**：密钥管理、密码管理（当前 `Auth` 改为 `认证`）。

标签样式：

- 默认态：次要文字色，无边框。
- 选中态：主色下划线 + 主色文字。
- hover 态：背景浅灰。

---

### 4.4 触发器标签页

#### 4.4.1 触发器列表

采用表格布局：

```text
事件        名称             命令                  操作
连接时      Test             ls                    ▶  编辑  ⋯
连接时      OnConnectTest    echo connected_ok     ▶  编辑  ⋯
```

**列说明**：

- 事件：使用 pill 标签，颜色按事件类型区分但不刺眼。
- 名称：普通正文。
- 命令：等宽字体 + 浅灰背景 code block，最大宽度截断，hover 显示完整命令 tooltip。
- 操作：
  - `▶` 播放图标：手动触发
  - `编辑`：文字按钮
  - `⋯` 更多菜单：删除、复制、查看日志

#### 4.4.2 添加触发器流程

1. 点击 `+ 添加触发器`。
2. 弹出抽屉/弹窗，左侧选择模板库或空白创建。
3. 右侧表单：
   - 事件类型（下拉）
   - 触发器名称（输入）
   - 命令/脚本（多行代码编辑器）
   - 高级选项：超时、错误是否继续、IP 重检间隔
4. 底部：`取消` `保存`。

#### 4.4.3 手动触发反馈

- 点击播放图标后，图标变为加载动画。
- 触发完成后显示临时成功/失败 toast：
  - 成功：`Test 已触发 · 耗时 120ms`
  - 失败：`Test 触发失败 · 退出码 1`

### 4.5 模板库弹窗

#### 4.5.1 弹窗结构

```text
+------------------------------------------+
| 模板库                    [+ 新建] [导入] [×]
+------------------------------------------+
| 搜索模板...                              |
+------------------------------------------+
| 模板列表              |  预览            |
|                       |                  |
| 更新防火墙白名单       |  #!/bin/bash     |
| (firewalld)            |  ...             |
| On IP Change      编辑|                  |
|                       |                  |
| 更新防火墙白名单       |  [使用此模板]    |
| (ufw)                |                  |
| On IP Change      编辑|                  |
|                       |                  |
| ...                   |                  |
+------------------------------------------+
```

#### 4.5.2 模板列表项

每项包含：

- 模板名称（主文本）
- 适用事件（次文本，使用事件标签）
- 操作：`编辑` 按钮 + `⋯` 菜单（删除、导出）

#### 4.5.3 预览面板

- 右侧或下方显示选中模板的脚本内容。
- 使用等宽字体 + 代码高亮（如支持）。
- 显示模板说明和变量占位符。
- 底部主按钮：`使用此模板`。

#### 4.5.4 新建/编辑模板

- 与添加触发器使用同一套表单组件，保持一致性。

### 4.6 设置弹窗/面板

#### 4.6.1 推荐结构：左侧导航 + 右侧内容

```text
+------------------------------------------+
| 设置                                  [×]
+------+-----------------------------------+
| 通用 | 语言                              |
| 代理 | 主题                              |
| 触发器| 开机自启                         |
| 通知 | 最小化到托盘                      |
| 关于 |                                   |
|      | 日志                              |
|      | 日志级别...                       |
|      | ...                               |
|      |                                   |
|      | [取消]        [保存]              |
+------+-----------------------------------+
```

#### 4.6.2 分类说明

| 分类 | 包含字段 |
|------|---------|
| 通用 | 语言、主题、开机自启、最小化到托盘、日志级别、日志写入文件、保留天数、最大文件大小 |
| 代理默认值 | 默认本地端口、默认测试 URL、代理类型 |
| 触发器默认值 | 默认超时、出错是否继续、IP 重检间隔 |
| 通知 | 连接成功、断开（非主动）、重连成功、认证失败 |
| 关于 | 版本号、检查更新、开源协议 |

#### 4.6.3 表单布局

- 标签左对齐，固定宽度 120px。
- 控件左对齐，统一宽度 240px 或 100%。
- 相关字段用分组标题 + 分割线组织。
- 复选框与说明文字纵向排列。

#### 4.6.4 操作栏

- 固定在面板底部。
- 左侧：`恢复默认`（谨慎操作，需二次确认）
- 右侧：`取消` `保存`

## 5. 交互流程优化

### 5.1 连接/断开流程

#### 5.1.1 当前问题

- 红色 × 让用户不敢断开。
- 断开操作没有反馈。

#### 5.1.2 重构后流程

1. 用户在详情页点击 `断开连接`（描边按钮）。
2. 按钮文案变为 `断开中...` 并禁用。
3. 成功后服务器状态变为灰色，侧边栏同步更新。
4. 失败时显示 toast：`断开失败 · 原因`。

### 5.2 删除服务器流程

1. 用户点击 `⋯` 菜单中的 `删除服务器`。
2. 弹出二次确认：
   ```
   确定删除服务器“3333”？
   删除后该服务器的触发器和代理配置将一并移除，无法恢复。
   
   [取消]  [删除]
   ```
3. 确认后删除，侧边栏移除该项，主区域回到空状态。

### 5.3 日志查看流程

1. 底部始终显示日志摘要行。
2. 点击 `展开` 后抽屉上滑，显示完整日志。
3. 日志支持：
   - 按级别筛选（全部 / INFO / WARN / ERROR）
   - 按来源筛选（全部 / network / proxy / trigger）
   - 搜索关键字并高亮
   - 一键复制全部日志
   - 清除日志（需确认）

### 5.4 模板使用流程

1. 用户在触发器标签页点击 `+ 添加触发器`。
2. 选择 `从模板库创建`。
3. 弹出模板库弹窗，右侧预览模板内容。
4. 点击 `使用此模板` 后，回到创建触发器表单，字段自动填充。
5. 用户可修改后保存。

## 6. 文案与语言规范

### 6.1 界面语言

- 默认界面语言为**简体中文**。
- 所有面向用户的文案、按钮、标签、提示均使用中文。
- 英文仅用于以下情况：
  - 技术缩写，如 `SSH`、`SOCKS5`、`HTTP`、`IP`、`URL`、`Auth`（但 Auth 在本方案中译为“认证”）。
  - 代码、命令、路径、变量名。
  - 日志中的原始输出。

### 6.2 事件类型命名

| 当前英文 | 建议中文 | 备注 |
|---------|---------|------|
| On Connect | 连接时 | 服务器连接成功后触发 |
| On Ip Change | IP 变化时 | 公网 IP 发生变化时触发 |
| On Process Dead | 进程结束时 | SSH 进程意外退出时触发 |
| On Reconnect | 重连时 | 每次重连尝试时触发 |
| On Port Closed | 端口关闭时 | 本地监听端口不可用时触发 |

如果必须保留英文，统一为：

- `On Connect`
- `On IP Change`
- `On Process Dead`
- `On Reconnect`
- `On Port Closed`

### 6.3 按钮文案规范

使用“动词 + 名词”结构，避免歧义：

| 不推荐 | 推荐 |
|--------|------|
| 删除 | 删除服务器 |
| 断开 | 断开连接 |
| 保存 | 保存设置 |
| 连接 | 连接服务器 |
| 添加 | 添加服务器 / 添加触发器 |

### 6.4 状态文案规范

| 状态 | 文案 |
|------|------|
| 已连接 | 已连接 |
| 连接中 | 连接中... |
| 已断开 | 已断开 |
| 连接失败 | 连接失败 · 原因 |
| 重连中 | 重连中（第 n 次） |

### 6.5 日志文案规范

- 日志前缀格式：`[2026-07-13T11:43:38.543Z] [WARN] [network] Network went offline`
- 级别：INFO、WARN、ERROR、DEBUG
- 来源：network、proxy、trigger、system
- 消息使用完整中文句子，避免 `logs.expand` 这类 key。

## 8. 实现优先级

### 8.1 P0 - 修正语义与安全感（立即执行）

| 任务 | 原因 |

| 底部日志抽屉化 | 减少固定占用空间，按需展开 |
